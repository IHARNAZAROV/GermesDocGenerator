# 1. Общая информация

## 1.1 Назначение программы

«Генератор договоров ГермесГарант» — это настольное Windows-приложение для риэлтеров агентства недвижимости **ООО ГермесГарант**. Программа автоматически создаёт пакет юридических документов на основе данных сделки, которые хранятся в файле Excel.

**Главная задача программы:** избавить риэлтера от ручного заполнения договоров. Достаточно один раз заполнить Excel-файл со всеми данными сделки — и программа сформирует любое количество готовых документов Word за несколько секунд.

## 1.2 Основные возможности

- **Загрузка данных из Excel** — программа читает готовый файл Excel с данными сделки и заполняет форму автоматически.
- **Редактирование данных прямо в программе** — все поля можно исправить без открытия Excel.
- **Сохранение изменений обратно в Excel** — отредактированные данные записываются в тот же файл или в новый.
- **Генерация 16 видов документов** одним нажатием кнопки:
  - Договоры оказания риэлтерских услуг (1, 2 или 3 собственника, общий или эксклюзивный)
  - Договор задатка
  - Договор на рекламные услуги
  - Договор о конвертации валюты
  - Договор хранения имущества
  - Соглашение о расторжении
  - Доверенность на представительство (ПНД)
  - Запрос на ПНД
  - Запрос в РСЦ
  - Согласие на обработку персональных данных
  - Расписка в получении ключей
  - Договор (физическое лицо — коммерческая структура)
- **Предварительный просмотр** документа перед сохранением.
- **Автоматический расчёт** суммы прописью (BYN и USD) и комиссии агентства.
- **Автоматическое склонение** ФИО в родительный и дательный падеж для корректного юридического текста.
- **Обновление шаблона** — добавление новых полей в Excel автоматически добавляет их в форму без изменения кода.

## 1.3 Требования к системе

| Параметр | Требование |
|---|---|
| Операционная система | Windows 10 / Windows 11 (64-бит) |
| Оперативная память | от 4 ГБ ОЗУ |
| Место на диске | от 300 МБ свободного места |
| Права пользователя | Стандартные (администратор не требуется для запуска) |
| Microsoft Word | **Не требуется** — документы создаются без Word |
| Microsoft Excel | **Не требуется** — программа читает .xlsx напрямую |

## 1.4 Запуск программы

### Через ярлык на рабочем столе

1. Дважды щёлкните по ярлыку **«Генератор договоров ГермесГарант»** на рабочем столе.
2. Подождите 3–5 секунд, пока откроется главное окно.
3. В строке состояния внизу появится надпись **«Готов к работе»** — программа готова.

### Через меню «Пуск»

1. Нажмите кнопку **«Пуск»** (значок Windows).
2. Найдите папку **«Генератор договоров ГермесГарант»**.
3. Щёлкните по пункту «Генератор договоров ГермесГарант».

### Через portable-версию (переносная)

1. Перейдите в папку с файлом программы (например, `D:\GermesGarant\`).
2. Дважды щёлкните по файлу **«Генератор договоров ГермесГарант-Portable.exe»**.

> **Примечание.** При первом запуске Windows может показать предупреждение «Защитник Windows SmartScreen». Нажмите «Подробнее» → «Выполнить в любом случае».

## 2.3 Левая панель — данные сделки

Левая панель занимает большую часть окна. Разделена на секции.

### Секция 1: Загрузка данных из Excel

Содержит:
- **Поле «Файл Excel»** — показывает полный путь к загруженному файлу. Только для чтения.
- **Строка успеха** — зелёная строка с галочкой и именем файла, появляется после успешной загрузки.

### Секция 2: Данные сделки

Центральная часть интерфейса. Делится на подсекции.

#### Подсекция «Объект недвижимости» (левая колонка)

Поля, описывающие продаваемый объект:

| Поле | Тип | Описание |
|---|---|---|
| Тип объекта | Текст | Например: квартира, дом, комната |
| Адрес | Текст | Полный адрес |
| Город | Текст | Город расположения |
| Улица | Текст | Название улицы |
| Дом | Текст | Номер дома |
| Корпус / Кв. | Пара полей | Два поля в одной строке через «/» — корпус и квартира |
| Этаж: из | Пара полей | Два поля — этаж и этажность (например: 5 из 9) |
| Кол-во комнат | Текст | Количество комнат |
| Общая площадь, м² | Текст | Общая площадь |
| Жилая площадь, м² | Текст | Жилая площадь |
| Площадь кухни, м² | Текст | Площадь кухни |
| Кадастровый № | Текст | Кадастровый номер объекта |
| Инвентарный № | Текст | Инвентарный номер (БТИ) |

#### Поля стоимости (под объектом, левая колонка)

| Поле | Тип | Описание |
|---|---|---|
| Цена, USD | Текст | Стоимость объекта в долларах |
| Цена, BYN | Текст (с валидацией) | Стоимость в белорусских рублях. Формат: `105000.50` или `105000,50` |
| Цена прописью | Только для чтения | Заполняется **автоматически** при вводе «Цена, BYN» |

> **Как работает «Цена прописью»:** как только вы вводите корректную сумму в поле «Цена, BYN», программа мгновенно вычисляет её словесное представление. Например, `105000` → «Сто пять тысяч белорусских рублей 00 копеек». Редактировать это поле вручную нельзя.

> **Ошибка формата BYN:** если вы ввели некорректную сумму (буквы, отрицательное число), под полем появится красная надпись «Введите корректную сумму», а поле «Цена прописью» останется пустым.

#### Подсекция «Продавец» (правая колонка)

Данные лица, которое продаёт объект (может действовать по доверенности):

| Поле | Тип | Описание |
|---|---|---|
| Фамилия | Текст | |
| Имя | Текст | |
| Отчество | Текст | |
| Дата рождения | Дата 📅 | |
| Паспорт серия | Текст | |
| Паспорт номер | Текст | |
| Идент. номер | Текст | Личный номер паспорта |
| Кем выдан | Текст | Организация, выдавшая паспорт |
| Дата выдачи | Дата 📅 | |
| Адрес регистрации | Текст | |
| Телефон | Текст | |
| Является собственником | Текст | Значение: `да` или `нет`. Влияет на доступность шаблонов! |
| Номер доверенности | Текст | Заполняется, если продавец не является собственником |
| Дата доверенности | Текст | Заполняется, если продавец не является собственником |

> **Важно: поле «Является собственником».** Это поле определяет, как программа подсчитывает количество собственников сделки, и влияет на то, какие шаблоны договоров доступны для выбора (подробнее — в разделе 4).

#### Подсекция «Покупатель»

Данные покупателя объекта. Поля аналогичны продавцу (ФИО, паспорт, дата рождения, адрес, телефон).

#### Подсекция «Продавцы / Собственники (до 3 человек)»

Содержит **три вкладки**: «Собственник 1», «Собственник 2», «Собственник 3».

**Как работают вкладки:**
- Нажмите на вкладку, чтобы переключиться.
- Активная вкладка подсвечена.
- Каждая вкладка содержит одинаковый набор полей (ФИО, паспорт, дата рождения, адрес, телефон, доля собственности), отображённых в **двух колонках** для компактности.

> **Поле «Доля собственности»** — указывается в виде дроби, например: `1/2`, `1/3`. Используется в договорах при долевой собственности.

> **Принцип подсчёта собственников:** программа автоматически определяет количество активных собственников по заполненности ключевых полей (Фамилия, Имя, Серия и номер паспорта, Идент. номер). Если поле «Является собственником» у продавца = `да`, то продавец считается собственником №1, а дополнительные собственники — это данные на вкладках.

#### Подсекция «Дополнительные данные»

Поля сделки, которые не вошли в основные блоки:

| Поле | Тип | Описание |
|---|---|---|
| Номер сделки | Текст | Внутренний номер |
| Дата договора | Дата 📅 | Дата подписания договора |
| Дата окончания договора | Дата 📅 | |
| Тип договора | Текст | |
| Эксклюзив | Текст | |
| Кол-во собственников | Текст | |
| Комиссия агентства | Только для чтения | Рассчитывается **автоматически** по тарифной таблице |
| Риэлтер | Текст | ФИО ответственного риэлтера |
| Задаток, BYN | Текст | |
| Задаток, USD | Текст | |
| Оплата услуг БТИ | Текст | |

> **Как работает «Комиссия агентства»:** поле вычисляется автоматически на основе «Цены BYN» по тарифной таблице Министерства юстиции РБ. Базовая величина = 45 BYN. Цена делится на базовую величину, по результату определяется процент (от 6% до 1.5%). Подробная таблица:

| Количество базовых величин | Процент |
|---|---|
| До 700 | 6,0% |
| Свыше 700 до 2000 | 5,0% |
| Свыше 2000 до 3000 | 4,0% |
| Свыше 3000 до 4200 | 3,0% |
| Свыше 4200 до 5000 | 2,5% |
| Свыше 5000 до 5800 | 2,4% |
| Свыше 5800 до 6600 | 2,3% |
| Свыше 6600 до 7500 | 2,2% |
| Свыше 7500 до 8300 | 2,1% |
| Свыше 8300 до 9100 | 2,0% |
| Свыше 9100 до 10000 | 1,9% |
| Свыше 10000 до 10500 | 1,8% |
| Свыше 10500 до 11600 | 1,7% |
| Свыше 11600 до 12400 | 1,6% |
| Свыше 12400 | 1,5% |

Пример: квартира за 200 000 BYN → 200 000 / 45 = 4 444 базовых величины → 3,0% → комиссия = 6 000 BYN.

## 2.4 Правая панель — шаблоны и настройки

### Секция 3: Выбор шаблонов договоров

Список документов, разделённый на три группы.

#### Группа «Основные договора риэлтерских услуг»

| Документ | Требуется собственников |
|---|---|
| Договор оказания риэлтерских услуг (1 собственник, общий) | 1 |
| Договор оказания риэлтерских услуг (1 собственник, эксклюзив) | 1 |
| Договор оказания риэлтерских услуг (2 собственника, общий) | 2 |
| Договор оказания риэлтерских услуг (2 собственника, эксклюзив) | 2 |
| Договор оказания риэлтерских услуг (3 собственника, общий) | 3 |
| Договор оказания риэлтерских услуг (3 собственника, эксклюзив) | 3 |
| Договор оказания риэлтерских услуг (физическое лицо — коммерческая структура) | Любое |

#### Группа «Договоры и соглашения»

- Договор задатка (стандартный)
- Договор на оказание рекламных услуг
- Договор о конвертации валюты
- Договор хранения имущества
- Соглашение о расторжении

#### Группа «Дополнительные документы»

- Доверенность на представительство (ПНД)
- Запрос на ПНД
- Запрос в РСЦ
- Согласие на обработку данных
- Расписка в получении ключей

#### Автоматическая блокировка шаблонов

Шаблоны с пометкой «1 собственник», «2 собственника», «3 собственника» **автоматически блокируются** в зависимости от количества заполненных собственников в форме. Заблокированные шаблоны:
- отображаются серым цветом
- галочку поставить невозможно
- при наведении курсора показывается подсказка: например, «Недоступно. В сделке участвуют два собственника.»

Шаблоны с пометкой «любое» (Договор задатка, Запрос в РСЦ и т.д.) всегда доступны.

#### Кнопки «Выбрать все» и «Снять все»

- **«Выбрать все»** — ставит галочки на всех **доступных** (незаблокированных) шаблонах.
- **«Снять все»** — снимает галочки со всех шаблонов (включая ранее выбранные).

### Секция 4: Настройки генерации

#### Поле «Папка для сохранения»

Показывает путь к папке, куда будут сохранены готовые документы.

- Если поле пустое — документы сохраняются в папку `output` внутри папки программы.
- Нажмите кнопку **«Обзор»** рядом, чтобы выбрать папку через диалоговое окно.

#### Флажок «Открыть файлы после создания»

- **Включён** (по умолчанию): после генерации каждый созданный документ автоматически открывается в программе по умолчанию (обычно Microsoft Word или LibreOffice Writer).
- **Выключен**: документы создаются, но не открываются автоматически.

#### Флажок «Добавить дату к имени файла»

- **Включён**: к имени файла добавляется дата создания. Например: `Договор задатка_17-07-2026.docx`.
- **Выключен** (по умолчанию): `Договор задатка.docx`.

## 2.11 Встроенный календарь для полей с датой

Поля типа «дата» (Дата договора, Дата рождения, Дата выдачи паспорта и т.д.) имеют встроенный календарь.

### Как открыть календарь:
- Нажмите на маленькую кнопку 📅 справа от поля, **или**
- Щёлкните по самому полю ввода.

### Как пользоваться календарём:
1. Стрелки **◀ ▶** переключают месяц.
2. Нажмите на **«Месяц Год»** в заголовке — откроется режим выбора месяца и года сразу.
3. Щёлкните по нужному числу — дата вставится в поле.
4. Кнопка **«Сегодня»** — вставляет сегодняшнюю дату.
5. Кнопка **«Очистить»** — очищает поле.
6. Клавиша **Esc** или клик вне календаря — закрывает без выбора.

### Ручной ввод даты:
Можно вводить дату вручную в формате **ДД.ММ.ГГГГ**. Программа автоматически добавляет точки:
- После ввода 2 цифр дня (например, `17`) автоматически добавляется точка → `17.`
- После ввода 2 цифр месяца (например, `17.07`) автоматически добавляется точка → `17.07.`
- Остаётся ввести год: `17.07.2026`

Выходные дни (суббота, воскресенье) в календаре отображаются другим цветом.

# 3. Порядок работы

## Полный цикл работы с программой

---

### Шаг 1. Запуск программы

1. Дважды щёлкните по ярлыку программы.
2. Дождитесь появления главного окна.
3. Убедитесь, что в строке состояния написано **«Готов к работе»**.

---

### Шаг 2. Загрузка Excel-файла с данными сделки

1. Нажмите кнопку **«Выбрать Excel»** на панели инструментов.
2. В открывшемся диалоге перейдите к папке с файлом сделки (например, `D:\Сделки\2026\Иванов\`).
3. Выберите нужный `.xlsx`-файл и нажмите **«Открыть»**.
4. На экране появится анимация загрузки («Чтение файла…»).
5. После загрузки под полем «Файл Excel» появится зелёная строка:  
   ✅ «Файл успешно загружен: Шаблон_Иванов.xlsx»
6. Все поля формы автоматически заполнятся данными из файла.

> **Если файл не загружается:** убедитесь, что он имеет расширение .xlsx и не открыт в другой программе (Excel). Смотрите раздел 7 «Возможные ошибки».

---

### Шаг 3. Проверка и редактирование данных

1. Просмотрите все заполненные поля.
2. При необходимости исправьте данные — щёлкните в нужное поле и отредактируйте.
3. Поля с изменениями отобразятся с **жёлтой рамкой** — это означает «несохранённые изменения».
4. Нажмите кнопку **«Проверить данные»** внизу левой панели, чтобы убедиться, что ключевые поля заполнены.

> **Совет:** проверьте вкладки собственников (Собственник 1, 2, 3). Убедитесь, что заполнены только нужные вкладки — лишние пустые вкладки на генерацию не влияют.

---

### Шаг 4. Сохранение изменений (если редактировали данные)

- **Если нужно сохранить в тот же файл:** нажмите кнопку **«Сохранить»** на панели инструментов.  
  Появится уведомление «✔ Изменения сохранены».
- **Если нужно создать новый файл:** нажмите **«Сохранить как…»** → выберите место и имя файла → нажмите «Сохранить».

> Сохранение изменений в Excel необязательно для генерации документов — генерация использует данные, которые сейчас видны в форме.

---

### Шаг 5. Настройка папки для сохранения документов

1. В правой панели найдите секцию **«4. Настройки генерации»**.
2. Нажмите кнопку **«Обзор»** и выберите папку, куда будут сохраняться готовые документы.
3. При необходимости включите/выключите:
   - «Открыть файлы после создания» — будут ли документы открываться автоматически.
   - «Добавить дату к имени файла» — будет ли к имени файла добавляться дата.

1. В правой панели найдите секцию **«3. Выбор шаблонов договоров»**.

3. Обратите внимание на заблокированные (серые) шаблоны — они недоступны для текущего количества собственников.
4. При необходимости нажмите **«Выбрать все»** или **«Снять все»**.

**Пример выбора для типичной сделки с 1 собственником:**
- ✅ Договор оказания риэлтерских услуг (1 собственник, общий)
- ✅ Договор задатка (стандартный)
- ✅ Согласие на обработку данных
- ✅ Расписка в получении ключей

### Шаг 7. Предварительный просмотр (необязательно)

1. Нажмите кнопку **«Предварительный просмотр»** в секции «5. Действия».
2. Дождитесь загрузки — появится окно предпросмотра.
3. Проверьте текст документа. Переключайтесь между вкладками для просмотра разных документов.

2. Программа начнёт создавать файлы по очереди.
3. Каждый успешно созданный файл отображается зелёным уведомлением.
4. Если включён флажок «Открыть файлы после создания» — каждый файл автоматически откроется в программе просмотра Word.

> **Где найти готовые файлы?** В папке, которую вы выбрали в шаге 5. Если папка не была выбрана — файлы находятся в подпапке `output` внутри папки программы.

Если данные для какого-либо плейсхолдера не заполнены, в документе на его месте будет **пустое место** (пустая строка). Программа не прерывает генерацию — она просто оставляет поле пустым. Пустые поля потребуется заполнить вручную в готовом документе.

# 5. Работа с Excel

## 5.1 Формат Excel-файла

Программа работает исключительно с файлами формата **.xlsx** (Excel 2007 и новее).

### Структура файла

- **Один лист** с именем «Сделка» (или любым — используется первый лист).
- **Столбец A** — названия блоков и полей.
- **Столбец B** — значения полей.

### Блоки данных

Файл разделён на **блоки** по следующей схеме:

| Заголовок блока (столбец A) | Что содержит |
|---|---|
| `СДЕЛКА` | Данные сделки (номер, дата, тип, цена и т.д.) |
| `ОБЪЕКТ` | Данные объекта недвижимости |
| `ПРОДАВЕЦ` | Данные продавца |
| `СОБСТВЕННИК №1` | Данные первого собственника |
| `СОБСТВЕННИК №2` | Данные второго собственника |
| `СОБСТВЕННИК №3` | Данные третьего собственника |
| `ПОКУПАТЕЛЬ` | Данные покупателя |

### Пример структуры

```
A                        B
───────────────────────────────
СДЕЛКА
Дата договора            15.07.2026
Номер сделки             2026-145
Стоимость USD            95000
Стоимость BYN            305000
...

ОБЪЕКТ
Тип объекта              квартира
Адрес                    г. Минск, ул. Ленина, д. 10, кв. 25
Этаж                     5
Этажность                9
...

ПРОДАВЕЦ
Фамилия                  Иванов
Имя                      Иван
Отчество                 Иванович
...
```

## 5.2 Как читается Excel

При нажатии «Выбрать Excel» программа:

1. Открывает первый лист файла.
2. Читает строки сверху вниз.
3. Когда встречает заголовок блока (`СДЕЛКА`, `ОБЪЕКТ` и т.д.) — запоминает, что последующие строки относятся к этому блоку.
4. Для каждой строки: значение из столбца A — это имя поля, значение из столбца B — значение поля.
5. Если ячейка B пуста — поле остаётся пустым в форме.
6. Даты из Excel автоматически конвертируются в формат **ДД.ММ.ГГГГ**.

> **Поле «Комиссия агентства»** никогда не читается из Excel — оно всегда вычисляется программой автоматически по тарифной таблице.

## 5.3 Поддерживаемые форматы ячеек

| Тип данных в Excel | Обработка программой |
|---|---|
| Текст | Читается как есть |
| Число | Конвертируется в строку |
| Дата | Конвертируется в формат ДД.ММ.ГГГГ |
| Формула | Берётся **результат** вычисления, не сама формула |
| Форматированный текст (Rich Text) | Форматирование теряется, читается только текст |
| Пустая ячейка | Поле остаётся пустым |

## 5.4 Как записывается Excel

При сохранении («Сохранить» / «Сохранить как…») программа:
1. Открывает исходный Excel-файл.
2. Изменяет **только ячейки столбца B**, соответствующие изменённым полям.
3. Всё форматирование, формулы, другие столбцы и листы **не затрагиваются**.
4. Сохраняет файл.

## 5.5 Возможные ошибки при работе с Excel

| Ошибка | Причина | Решение |
|---|---|---|
| «Файл не содержит листов» | Excel-файл пустой или повреждён | Проверьте файл — откройте его вручную в Excel |
| «Ошибка при чтении файла» | Файл не является корректным .xlsx, или открыт в Excel | Закройте файл в Excel и попробуйте снова |
| «Файл прочитан, но данные не получены» | Структура файла не соответствует ожидаемой | Убедитесь, что в файле есть блоки `СДЕЛКА`, `ОБЪЕКТ` и т.д. |
| Поля остались пустыми после загрузки | В файле нет соответствующего блока, или имена полей не совпадают | Проверьте, что заголовки блоков написаны точно так же, как указано выше |

# 6. Генерация документов

## 6.1 Как происходит создание документа

1. Программа собирает **все данные из полей формы** в единый объект данных.
2. Автоматически вычисляет производные значения (ФИО в падежах, суммы прописью, остаток после задатка и т.д.).
3. Читает шаблон .docx из папки `templates/working/`.
4. Заменяет плейсхолдеры `{{...}}` реальными значениями.
5. Сохраняет готовый .docx-файл в папку назначения.
6. Если включён флажок «Открыть файлы после создания» — открывает файл в программе просмотра.

## 6.2 Какие данные используются

Используются **данные, которые сейчас видны в форме** — не обязательно из Excel. Если вы что-то изменили в форме, но не сохранили в Excel, в документ попадут ваши изменения.

### Автоматически вычисляемые данные:

- **ФИО в родительном падеже** (Кого?) — Иванова Ивана Ивановича
- **ФИО в дательном падеже** (Кому?) — Иванову Ивану Ивановичу
- **Дата прописью** — «пятнадцатое июля две тысячи двадцать шестого года»
- **Дата в длинном формате** — «15 июля 2026»
- **Орган выдачи паспорта в творительном падеже** — «Лидским РОВД» вместо «Лидский РОВД»
- **Тип объекта в родительном падеже** — «квартиры» вместо «квартира»
- **Стоимость прописью BYN** — «Триста пять тысяч белорусских рублей 00 копеек»
- **Стоимость прописью USD** — «Девяносто пять тысяч»
- **Сумма задатка прописью**
- **Остаток стоимости = Цена USD − Задаток USD**
- **Данные риэлтера** — если в поле «Риэлтер» введена фамилия из справочника, программа автоматически подставляет полные данные (ФИО, номер аттестата, номер карточки, срок действия).

Программа содержит встроенный справочник сотрудников:

|---|---|---|---|
| Турко | Турко Ольга Ростиславовна | 1931 | АВ 0028488 |
| Юшкевич | Юшкевич Наталья Вячеславовна | 1839 | АВ 0028486 |
| Москаленко | Москаленко Екатерина Викторовна | 2030 | АВ 0028487 |
| Эйсмонт | Эйсмонт Ольга Ивановна | 2448 | АВ 0028485 |

Введите фамилию в поле «Риэлтер» (на русском или латинском) — и данные агента автоматически подставятся во все шаблоны.

## 6.3 Куда сохраняется документ

| Ситуация | Папка назначения |
|---|---|
| Папка выбрана в настройках | Выбранная папка |
| Папка **не** выбрана | `[папка программы]\output\` |

Если папка назначения не существует — программа создаёт её автоматически.

### Имена создаваемых файлов:

| Флажок «Добавить дату» | Пример имени |
|---|---|
| Выключен | `Договор задатка.docx` |
| Включён | `Договор задатка_17-07-2026.docx` |

## 6.4 Как изменить путь сохранения

1. В секции «4. Настройки генерации» нажмите кнопку **«Обзор»**.
2. В диалоге выберите или создайте нужную папку.
3. Нажмите **«ОК»** — путь отобразится в поле «Папка для сохранения».

> Можно также ввести путь к папке вручную прямо в поле «Папка для сохранения».

## 6.5 Что делать при ошибках генерации

1. Прочитайте текст ошибки во всплывающем уведомлении или в баннере.
2. Типичные причины:
   - Шаблон .docx отсутствует или повреждён — проверьте папку `templates/working/`.
   - Нет прав на запись в папку назначения — выберите другую папку.
   - Документ с таким именем уже открыт в Word — закройте его.

# 8. Часто задаваемые вопросы (FAQ)

### В: Программа не открывается. Что делать?

**О:** Попробуйте следующее:
1. Убедитесь, что Windows не заблокировала файл программы (правой кнопкой по .exe → Свойства → «Разблокировать»).
2. Запустите программу от имени администратора (правая кнопка → «Запустить от имени администратора»).
3. Проверьте, что в папке программы есть все файлы (она не была неполностью скопирована).

---

### В: Нажал «Выбрать Excel», выбрал файл, но форма осталась пустой.

**О:** Скорее всего, структура файла не соответствует шаблону. Убедитесь, что файл содержит заголовки блоков (`СДЕЛКА`, `ОБЪЕКТ`, `ПРОДАВЕЦ` и т.д.) в столбце A. Используйте официальный шаблон сделки агентства.

---

### В: Цена прописью не заполняется автоматически.

**О:** Убедитесь, что в поле «Цена, BYN» введено корректное число. Если под полем есть красная надпись «Введите корректную сумму» — исправьте формат ввода (используйте точку, а не запятую для дробной части, либо запятую — оба варианта поддерживаются).

---

### В: Кнопка «Сохранить» не активна (серая).

**О:** Кнопка становится активной только тогда, когда:
1. Загружен Excel-файл, И
2. В форме есть несохранённые изменения (хотя бы одно поле изменено).

Если файл загружен, но вы ничего не меняли — используйте «Сохранить как…», которая всегда доступна при загруженном файле.

---

### В: Шаблоны для 2 и 3 собственников недоступны (серые).

**О:** Это нормальное поведение. Программа автоматически определяет количество собственников и блокирует неподходящие шаблоны. Для активации шаблонов на 2 собственников — заполните данные на вкладке «Собственник 2» (или «Собственник 1» и «Собственник 2» если продавец не является собственником).

---

### В: Как сформировать документы без загрузки Excel-файла?

**О:** Это возможно — данные можно ввести вручную прямо в форму. Просто начните заполнять поля. Кнопка «Сформировать документы» работает с любыми данными в форме, независимо от того, загружен файл или нет.

---

### В: После нажатия «Сформировать документы» ничего не произошло.

**О:** Убедитесь, что хотя бы один шаблон отмечен галочкой. Если галочки не стоят — документы не создаются.

---

### В: Документ создался, но некоторые поля в нём пустые.

**О:** Соответствующие данные не были заполнены в форме. Заполните нужные поля и сформируйте документ повторно. Или отредактируйте готовый .docx-файл вручную в Word.

---

### В: Можно ли изменить шаблон договора?

**О:** Да, если у вас есть Microsoft Word и доступ к папке `templates/working/`. Откройте файл .docx, внесите изменения в текст (не трогая плейсхолдеры `{{...}}`), сохраните. При следующей генерации будет использован обновлённый шаблон.

---

### В: Как добавить нового риэлтера в справочник?

**О:** Отредактируйте файл `config/agents.json` и файл `js/agents-config.js` — добавьте запись о новом сотруднике (ФИО, аттестат, карточка, ключевые слова для поиска). Перезапустите программу.

---

### В: Предпросмотр показывает текст не так, как в готовом документе.

**О:** Это ожидаемое поведение. Предпросмотр показывает только **текст** документа без форматирования Word (таблицы, шрифты, отступы, жирный текст и т.д.). Итоговый файл .docx будет отформатирован корректно.

---

### В: Как узнать, куда сохранился документ?

**О:** Посмотрите поле «Папка для сохранения» в секции «4. Настройки генерации». Если оно пустое — документы находятся в папке `output` внутри папки программы.

# 9. Советы по работе

### 💡 Совет 1. Заполняйте Excel-файл заранее

Заведите привычку заполнять Excel-шаблон сделки полностью до открытия программы. Тогда загрузка займёт несколько секунд, а вам останется только выбрать шаблоны и нажать «Сформировать».

---

### 💡 Совет 2. Используйте «Проверить данные» перед генерацией

Нажмите кнопку «Проверить данные» перед тем, как нажать «Сформировать документы». Это займёт секунду, но убережёт от договора с пустой датой или без фамилии.

---

### 💡 Совет 3. Используйте предпросмотр для проверки сложных сделок

Перед окончательной генерацией нажмите «Предварительный просмотр» и бегло проверьте текст — особенно ФИО, паспортные данные, суммы. Это быстрее, чем открывать Word.

---

### 💡 Совет 4. Включайте «Добавить дату к имени файла»

Если вы часто переформировываете документы по одной сделке (например, после исправлений), включите флажок «Добавить дату к имени файла». Это поможет различать версии: `Договор задатка_15-07-2026.docx` и `Договор задатка_17-07-2026.docx`.

---

### 💡 Совет 5. Создавайте папку для каждой сделки

Перед генерацией создайте отдельную папку для сделки (например, `D:\Сделки\2026\Иванов\`) и выберите её как папку сохранения. Тогда все документы по сделке будут в одном месте.

---

### 💡 Совет 6. Сохраняйте изменения в Excel сразу

Если вы что-то исправили в форме, нажмите «Сохранить» сразу — не откладывайте. Иначе при случайном закрытии программы данные потеряются.

---

### 💡 Совет 7. Правильно заполняйте «Является собственником»

Это критически важное поле. Если продавец является собственником — введите `да`. Если действует по доверенности — введите `нет` и заполните вкладку «Собственник 1». Иначе доступность шаблонов будет определена неверно.

---

### 💡 Совет 8. Вводите цену в обоих полях

Заполните и «Цена, USD», и «Цена, BYN» — разные шаблоны используют разные поля. Поле «Цена прописью» заполнится автоматически.

---

### 💡 Совет 9. Не редактируйте шаблоны, пока они открыты в программе

Если вы редактируете шаблон .docx в Word и одновременно запускаете генерацию в программе — файл может быть заблокирован. Сначала закройте Word, потом генерируйте.

---

### 💡 Совет 10. Резервное копирование Excel-файлов

Храните копии Excel-файлов сделок в надёжном месте (сетевая папка, облако). Программа умеет «Сохранить как…» — используйте эту функцию для создания резервных копий с новым именем.

## 11.4 Рекомендации по резервному копированию

### Что нужно регулярно копировать:

1. **Excel-файлы сделок** (`*.xlsx`) — основные данные. Рекомендуется хранить в сетевой папке или облаке.

2. **Папку `templates/working/`** — рабочие шаблоны Word. Если шаблон случайно испорчен, его можно восстановить из папки `templates/Original/`.

3. **Файл `fields-config.json`** и **`js/fields-config.js`** — если вы обновляли шаблон через кнопку «Обновить шаблон».

4. **Файл `config/agents.json`** и **`js/agents-config.js`** — если вы добавляли новых риэлтеров.

### Рекомендуемая периодичность:
- Файлы сделок — ежедневно (автоматическое резервное копирование Windows).
- Папка шаблонов — после каждого изменения шаблонов.
- Конфигурационные файлы — после каждого изменения.

### Как восстановить шаблон из оригинала:

1. Откройте папку `templates/Original/` — здесь хранятся нетронутые оригинальные шаблоны.
2. Скопируйте нужный файл.
3. Вставьте в папку `templates/working/`, заменив повреждённый файл.

# 12. Предложения по улучшению интерфейса

В ходе анализа приложения выявлены следующие аспекты UX/UI, которые могут вызвать затруднения у пользователей.

---

### 12.1 Отсутствие горячих клавиш для основных операций

**Проблема:** Нет горячих клавиш для часто используемых действий (Ctrl+S — Сохранить, Ctrl+Shift+S — Сохранить как, F5 — Сформировать документы, Ctrl+O — Открыть файл).

**Предложение:** Добавить стандартные сочетания клавиш для всех кнопок панели инструментов и показывать их подсказки в тултипах кнопок.

---

### 12.2 Поле «Является собственником» требует точного текста

**Проблема:** Поле «Является собственником» принимает только значения `да` / `нет` (строго в нижнем регистре). Пользователь, написавший `Да` с заглавной буквы, может получить неверное поведение. Поле никак не подсказывает допустимые значения.

**Предложение:** Заменить текстовое поле на выпадающий список (`<select>`) с вариантами «Да» / «Нет», или хотя бы добавить плейсхолдер «да / нет» и нормализацию регистра при чтении.

---

### 12.3 Кнопка «Обновить шаблон» вынесена на основную панель инструментов

**Проблема:** Кнопка «Обновить шаблон» — административная операция, которую рядовой пользователь никогда не должен нажимать. Её случайное нажатие перезагружает приложение и сбрасывает несохранённые данные.

**Предложение:** Скрыть кнопку в меню «Файл» → «Администрирование» или добавить диалог подтверждения: «Обновление шаблона перезагрузит приложение. Все несохранённые данные будут потеряны. Продолжить?»

---

### 12.4 Нет индикатора прогресса при генерации нескольких документов

**Проблема:** При генерации 10+ документов пользователь не видит, сколько уже создано и сколько осталось. Программа кажется «замёрзшей».

**Предложение:** Добавить прогресс-бар или счётчик «Создание документа 3 из 10…» в строке состояния или в виде overlay.

---

### 12.5 Нет возможности открыть папку с готовыми документами прямо из программы

**Проблема:** После генерации пользователь должен самостоятельно открыть проводник и перейти в папку с документами.

**Предложение:** Добавить кнопку «Открыть папку» рядом с полем «Папка для сохранения» — она открывала бы папку в проводнике Windows (аналогично тому, как уже работает функция `shell.openPath`).

---

### 12.6 Отсутствует кнопка «Очистить» для поля с датой без открытия календаря

**Проблема:** Чтобы очистить поле с датой, нужно либо открыть календарь и нажать «Очистить» внутри него, либо вручную стереть текст. Нет удобного способа сбросить дату одним щелчком.

**Предложение:** Добавить небольшую кнопку «×» справа от поля с датой (по аналогии с современными UI-библиотеками) — для быстрой очистки без открытия календаря.

---

### 12.7 Кнопка «Проверить данные» проверяет только 5 полей

**Проблема:** Кнопка «Проверить данные» проверяет только «Дата договора», «Тип договора», «Адрес», «Фамилия продавца» и «Фамилия покупателя». Пользователь может решить, что всё заполнено, но упустить важные паспортные данные.

**Предложение:** Расширить проверку до всех полей, которые требуются выбранными шаблонами. Или добавить отдельную кнопку «Глубокая проверка», которая покажет полный список пустых полей.

---

### 12.8 Нет подтверждения перед «Очистить форму»

**Проблема:** Нажатие кнопки «Очистить форму» немедленно сбрасывает все данные без запроса подтверждения. Случайное нажатие приводит к потере всех данных.

**Предложение:** Добавить диалог подтверждения: «Все данные в форме будут удалены. Продолжить?» с кнопками «Да» / «Отмена».

---

### 12.9 Поле «Папка для сохранения» не запоминается между сессиями

**Проблема:** При каждом новом запуске программы поле «Папка для сохранения» пустое. Пользователь вынужден каждый раз выбирать папку заново.

**Предложение:** Сохранять последнюю выбранную папку в локальных настройках (`localStorage` или файл настроек) и восстанавливать при следующем запуске.

---

### 12.10 Нет информации о риэлтере, распознанном из справочника

**Проблема:** Когда в поле «Риэлтер» вводится фамилия и программа автоматически подставляет данные из справочника — пользователь не видит, что это произошло. Нет визуального подтверждения, что агент распознан.

**Предложение:** Добавить небольшой индикатор под полем «Риэлтер» при успешном распознавании: например, «✔ Распознан: Турко Ольга Ростиславовна, аттестат №1931».
