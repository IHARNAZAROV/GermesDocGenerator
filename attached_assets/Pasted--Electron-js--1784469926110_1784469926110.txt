Ты — ведущий инженер-разработчик на Electron.js с глубоким пониманием жизненного цикла приложений и работы с файловой системой. 

### Задача
Напиши законченный модуль обновления (модуль автоапдейта) для **Portable-версии** Electron-приложения на **Vanilla JavaScript** (ES6+), без использования сторонних фреймворков и тяжелых библиотек сборщиков. Система должна синхронизироваться с **GitHub Releases**.

### Контекст и Ограничения (Важно)
1. **Специфика Portable:** Приложение не установлено в систему, оно может находиться в любой папке (например, на флешке). Стандартный `electron-updater` для NSIS/AppImage здесь не подходит, так как нам нужно обновить файлы «на месте» или перезаписать текущую директорию приложения.
2. **Стек:** Electron (Main и Renderer процессы), чистый JavaScript (Vanilla JS), встроенные модули Node.js (`fs`, `path`, `child_process`, `crypto`), модуль `electron` (для `net` или использовать встроенный `fetch`).

### Сценарий работы системы (Workflow)
1. **Инициализация:** При старте приложения в Main-процессе запускается проверка.
2. **Проверка версии:** Приложение делает запрос к GitHub Releases API (`https://api.github.com/repos/{owner}/{repo}/releases/latest`) и сравнивает `tag_name` с текущей версией приложения (`app.getVersion()`).
3. **Уведомление пользователя:** Если найдена новая версия, Main-процесс отправляет событие в Renderer-процесс. Renderer показывает кастомное модальное окно (HTML/CSS) с вопросом: "Доступна новая версия X.X.X. Обновиться?".
4. **Скачивание:** При согласии пользователя Renderer отправляет сигнал обратно. Main-процесс начинает скачивание zip/exe архива новой версии с GitHub, отображая прогресс-бар в интерфейсе.
5. **Распаковка и Замена (Стратегия Portable):** 
   - Файл скачивается во временную директорию (`app.getPath('temp')`).
   - Поскольку запущенный `.exe` файл (или бинарник на Linux/macOS) нельзя перезаписать, пока он работает, нужно сгенерировать и запустить отдельный легковесный скрипт/батник (`.bat` для Windows или `.sh` для POSIX) через `child_process.spawn`, после чего немедленно закрыть текущее приложение (`app.quit()`).
   - Скрипт-помощник должен: подождать 1-2 секунды (пока Electron закроется), распаковать/переместить новые файлы в исходную папку приложения с заменой старых, запустить обновленный исполняемый файл и удалить сам себя.

### Что необходимо сгенерировать:
1. **`updater.js` (Main Process):** Логика запроса к GitHub API, скачивания файла через `electron.net` или `https`, создание временного скрипта для перезаписи файлов и закрытие приложения.
2. **`preload.js`:** Безопасный мост `contextBridge` для связи между процессами (передача событий `update-available`, `download-progress` и отправка ответа пользователя `start-update`).
3. **Пример UI (Renderer):** Простой HTML/JS код для отображения диалогового окна обновления и прогресса скачивания на чистом Vanilla JS.
4. **Скрипт миграции:** Код генерации `.bat` / `.sh` файла, который выполняет физическую замену файлов после закрытия процесса Electron.

Код должен быть максимально чистым, модульным, снабжен понятными комментариями на русском языке и обрабатывать основные ошибки (отсутствие интернета, ошибка записи в папку). Применяй безопасные практики Electron (без `remote`, с включенным `contextIsolation`).
