Необходимо выполнить рефакторинг интерфейса приложения, добавив современную систему микроанимаций, уведомлений и обратной связи (Micro Interactions & Feedback), чтобы интерфейс ощущался живым, отзывчивым и современным.

Главная цель — создать ощущение постоянной обратной связи с пользователем, аналогично современным desktop-приложениям (Figma, Cursor, JetBrains IDE, Microsoft 365, Arc Browser, Notion, Linear).

⚠️ Не изменять существующую бизнес-логику приложения. Все изменения должны касаться только UI/UX.

Общие требования

Каждое действие пользователя должно сопровождаться понятной визуальной реакцией.

Не использовать MessageBox для успешных операций.

Использовать современные Toast Notifications, Status Messages, Progress Overlay и микроанимации.

Все уведомления должны быть ненавязчивыми.

Продолжительность анимаций:

Fade — 150–200 ms
Slide — 180–250 ms
Hover — 120–150 ms
1. Импорт Excel

После выбора Excel-файла:

кнопка "Выбрать Excel" временно переходит в состояние загрузки;
отображается индикатор импорта;
после завершения появляется зелёный Toast.

Например:

✔ Excel успешно загружен

Файл:
Продажа_квартиры.xlsx

Импортировано:
128 полей

Требуют проверки:
4 поля

Toast автоматически скрывается через 4–5 секунд.

Если импорт выполнен с предупреждениями:

⚠ Excel загружен

Импортировано:
122 поля

Не удалось определить:

• Телефон покупателя
• Стоимость прописью
• Дата доверенности

[Перейти]

Кнопка "Перейти" автоматически открывает первое проблемное поле.

2. Выбор документов

При выборе шаблона документа:

вместо простой установки галочки показать Toast:

✓ Договор купли-продажи добавлен

При снятии галочки:

Документ удалён из списка формирования

Если пользователь выбрал несколько документов подряд:

Добавлено 3 документа
3. Генерация документов

Во время генерации документов отображать Overlay Progress.

Например:

Формирование документов

██████░░░░

3 из 8

Создание договора...

Создание акта...

Создание приложения...

После завершения:

✔ Документы успешно сформированы

Создано:
8 документов

[Открыть папку]

[Открыть документ]
4. Автосохранение

В статус-баре отображать состояние.

Во время сохранения:

● Сохранение...

После окончания:

✓ Все изменения сохранены

11:42:15

Не использовать всплывающее окно.

5. Ручное сохранение

После успешного сохранения проекта показать Toast:

✓ Проект сохранён

11:42:15
6. Изменение данных

При изменении важных полей отображать краткое уведомление.

Например:

Стоимость обновлена

или

Изменения сохранены автоматически

Не показывать уведомление при каждом вводе символа.

Показывать только после завершения изменения значения.

7. Проверка заполнения

При запуске проверки отображать Progress.

После окончания:

Проверка завершена

Ошибок:
2

Предупреждений:
5
8. Очистка формы

После очистки показать:

Форма очищена

[Отменить]

Кнопка "Отменить" доступна 5 секунд.

9. Undo

После действий:

очистка формы;
удаление документа;
удаление строки;
удаление данных;

показывать Toast:

Действие выполнено

[Отменить]
10. Выбор риэлтера

После выбора другого риэлтера показать:

Активный риэлтер изменён

Ольга Турко
11. Выбор типа объекта

После выбора типа объекта показать:

Тип объекта:

Дом

Добавлены рекомендации по заполнению.
12. Smart Tips

При изменении типа объекта карточка Smart Tips должна плавно обновляться.

Использовать Fade Transition.

Не менять содержимое мгновенно.

13. Переход к ошибке

При переходе к ошибке:

плавная прокрутка;
автоматическое раскрытие раздела;
установка фокуса;
кратковременная подсветка поля.
14. Заполнение обязательного поля

После заполнения обязательного поля:

индикатор рядом меняется

○

↓

✓

с небольшой анимацией.

15. Завершение заполнения

Когда заполнены все обязательные поля:

показать Toast:

🎉 Все обязательные поля заполнены.

Можно формировать документы.
16. Ошибки

Ошибки должны отображаться в современном виде.

Например:

Не удалось сохранить проект.

Проверьте права доступа.

[Повторить]

[Подробнее]

Не использовать стандартные MessageBox, если это не критическая ошибка.

17. Hover-анимации

Добавить лёгкие микроанимации для всех интерактивных элементов:

кнопок;
карточек;
строк списка;
пунктов меню;
документов;
элементов навигации.

Использовать:

небольшое увеличение (до 102%);
плавное изменение цвета;
лёгкую тень.

Без чрезмерных эффектов.

18. Пустые состояния

Если отсутствуют данные:

вместо пустого экрана показывать информативную карточку.

Например:

Excel ещё не выбран.

Выберите файл для начала работы.

[Выбрать Excel]

Если документы не выбраны:

Документы не выбраны.

Отметьте шаблоны для формирования.
19. Последовательное сопровождение пользователя

После каждого успешно выполненного этапа показывать следующую рекомендацию.

Например:

После импорта Excel:

Следующий шаг:

Проверьте сведения об объекте.

После проверки объекта:

Отлично.

Теперь заполните данные продавца.

После продавца:

Следующий шаг:

Проверьте покупателя.

После заполнения всех обязательных данных:

Все проверки завершены.

Можно приступать к формированию документов.
20. Общие требования
Все уведомления должны использовать единый стиль.
Все Toast должны появляться в одном месте интерфейса.
Одновременно отображать не более трёх уведомлений.
Новые уведомления должны заменять старые без визуального "скачка".
Все микроанимации должны быть плавными и не отвлекать пользователя.
Не блокировать работу пользователя во время отображения уведомлений.
Все существующие функции приложения должны продолжать работать без изменений.

После реализации провести рефакторинг новой подсистемы уведомлений, устранить дублирование кода и использовать единый компонент Toast/Notification Center, чтобы любые новые уведомления в будущем можно было добавлять через общий API, а не отдельной реализацией в каждом модуле.
