Необходимо создать профессиональную инструкцию пользователя для настольного приложения Windows.

Перед написанием инструкции внимательно изучи весь проект, включая:

- структуру проекта;
- исходный код;
- интерфейс приложения;
- все окна;
- все кнопки;
- все поля ввода;
- все вкладки;
- все диалоговые окна;
- все настройки;
- весь пользовательский сценарий;
- сообщения об ошибках;
- всплывающие подсказки;
- работу с файлами;
- создание документов;
- импорт и экспорт данных;
- используемые шаблоны Word и Excel.

После полного анализа приложения составь максимально подробную инструкцию пользователя.

Инструкция должна быть написана простым, понятным языком и ориентирована на пользователя, который впервые открыл программу.

Структура инструкции должна включать:

# 1. Общая информация
- назначение программы;
- основные возможности;
- требования к системе;
- запуск программы.

# 2. Интерфейс программы
Подробно описать каждый элемент интерфейса:
- главное окно;
- панели;
- кнопки;
- поля ввода;
- выпадающие списки;
- вкладки;
- строки состояния;
- индикаторы.

Для каждого элемента необходимо объяснить:
- назначение;
- принцип работы;
- когда используется;
- возможные ограничения.

# 3. Порядок работы

Пошагово описать полный цикл работы пользователя:

от запуска программы до получения готового документа.

Каждый шаг оформить отдельно.

Например:

Шаг 1.
...

Шаг 2.
...

Шаг 3.
...

И так далее.

# 4. Работа с шаблонами

Подробно описать:

- как выбираются шаблоны;
- какие шаблоны поддерживаются;
- где находятся;
- как добавить новый шаблон;
- какие поля используются;
- что происходит при отсутствии обязательных данных.

# 5. Работа с Excel

Описать:

- как загружается Excel;
- какие листы используются;
- какие поля считываются;
- какие ошибки могут возникнуть;
- как исправить ошибки.

# 6. Генерация документов

Подробно описать:

- как происходит создание документа;
- какие данные используются;
- куда сохраняется документ;
- как изменить путь сохранения;
- что делать при ошибках.

# 7. Возможные ошибки

Для каждой ошибки необходимо указать:

- текст ошибки;
- возможную причину;
- способы решения.

# 8. Часто задаваемые вопросы (FAQ)

Добавить раздел с наиболее вероятными вопросами пользователей.

# 9. Советы по работе

Дать рекомендации, которые помогут работать быстрее и избежать ошибок.

# 10. Горячие клавиши

Если приложение поддерживает сочетания клавиш — описать их.

# 11. Приложение

Добавить:

- описание структуры каталогов;
- описание создаваемых файлов;
- список используемых форматов;
- рекомендации по резервному копированию.

------------------------------------

Требования к оформлению:

- использовать Markdown;
- использовать заголовки H1-H3;
- использовать таблицы там, где это уместно;
- использовать нумерованные списки;
- использовать маркированные списки;
- выделять важные замечания;
- приводить примеры использования;
- описывать каждое действие пользователя максимально подробно.

Если в приложении имеются функции, которые неочевидны, обязательно объяснить принцип их работы.

Если обнаружены недоработки интерфейса или потенциально непонятные места для пользователя, после инструкции создать отдельный раздел:

# Предложения по улучшению интерфейса

В нем перечислить все найденные проблемы UX/UI и предложить варианты их улучшения.

Цель — создать полноценное руководство пользователя уровня коммерческого программного обеспечения объемом 20–50 страниц, не упуская ни одной функции приложения.
