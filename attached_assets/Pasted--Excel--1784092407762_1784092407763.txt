Проведи полный анализ Excel-шаблона, который используется для генерации документов. Необходимо сделать систему максимально масштабируемой, чтобы при изменении шаблона не приходилось переписывать код приложения.

Задачи
1. Проанализировать шаблон Excel
Просканировать все листы Excel.
Найти все используемые плейсхолдеры (например: {{clientName}}, {{price}}, ${contractNumber} или другой используемый формат).
Определить:
на каком листе расположен плейсхолдер;
адрес ячейки;
название плейсхолдера;
используется ли он несколько раз;
имеются ли объединённые ячейки.
2. Создать JSON-словарь плейсхолдеров

Необходимо автоматически сформировать единый словарь всех найденных полей.

Например:

{
  "clientName": {
    "sheet": "Договор",
    "cells": ["B12", "F4"],
    "type": "text"
  },
  "sellerName": {
    "sheet": "Договор",
    "cells": ["B18"],
    "type": "text"
  },
  "contractPrice": {
    "sheet": "Расчет",
    "cells": ["D15"],
    "type": "number"
  }
}

Данный JSON должен использоваться как единственный источник информации о полях шаблона.

3. Реализовать динамическую генерацию формы

Форма приложения не должна содержать жёстко прописанные поля.

Необходимо:

строить форму автоматически по JSON-словарю;
название поля брать из словаря;
предусмотреть возможность отображения пользовательского названия (label);
поддерживать группировку полей;
предусмотреть возможность скрытия отдельных полей через конфигурацию.
4. Простое добавление новых полей

Добавление нового поля должно состоять только из следующих действий:

Добавить новый плейсхолдер в Excel.
Пересканировать шаблон.
JSON автоматически обновляется.
Поле автоматически появляется в интерфейсе.
Генерация документа начинает использовать его без изменения исходного кода.

Добавление нового поля не должно требовать изменения JavaScript, HTML или другой бизнес-логики.

5. Автоматическое удаление отсутствующих полей

Если какое-либо поле отсутствует в Excel-шаблоне:

оно не должно отображаться в окне приложения;
его нельзя редактировать;
его нельзя отправить на генерацию документа;
оно автоматически исключается из формы после очередного анализа шаблона.

Интерфейс приложения всегда должен полностью соответствовать текущему Excel-шаблону.
